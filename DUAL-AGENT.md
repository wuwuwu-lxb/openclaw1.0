## 🔄 协作机制

### 1. 状态汇报（定期）

**频率：** 每 4-6 小时一次

**本地 → 服务器：**
- 用户在线状态
- 今日完成事项
- 当前任务进度
- 用户情绪/状态备注

**服务器 → 本地：**
- 定时任务执行情况
- 网络信息收集结果
- 服务器运行状态
- 提醒事项

### 2. 信息同步（实时）

**同步内容：**
- 主人重要决策/计划变更
- 新增待办事项
- 完成的任务
- 重要对话上下文

**同步方式：** 飞书消息互相发送

### 3. 任务分工

| 任务类型 | 负责方 | 说明 |
|---------|-------|------|
| 公众号写稿 | 服务器 | 周日 20:00 执行，利用网络优势搜索资料 |
| 日常检查 | 服务器 | 22:00 检查待办、生成建议 |
| 文件整理 | 本地 | C 盘、桌面、下载文件夹整理 |
| 即时对话 | 本地 | 用户直接交互 |
| 长期监控 | 服务器 | 网站可用性、API 状态等 |
| 学习陪伴 | 本地 | 高数/线代/C 语言等学习辅助 |

### 4. 互相讨论

**场景：**
- 复杂问题决策（双端各自分析后讨论）
- 主人状态评估（双方视角综合判断）
- 任务优化建议（互相提出改进方案）

**方式：** 飞书私聊或群聊
